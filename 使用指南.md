# CPU_YELLOW 使用指南

## 项目概述

CPU_YELLOW 是一款兼容 MIPS32 指令系统的五级流水线处理器设计项目。处理器支持 57 条 MIPS 指令，具备完整的数据前递、冒险检测、分支预测、CP0 例外处理等功能，采用经典的五级流水线结构：取指(IF) → 译码(ID) → 执行(EX) → 访存(MEM) → 写回(WB)。

本项目使用 Verilog（RTL 设计）和 SystemVerilog（仿真测试平台）编写，支持 Verilator 和 Icarus Verilog 两种仿真工具。

## 环境要求

- **Linux / WSL / macOS** 或 Windows（需安装对应工具）
- **Verilator** (≥ 4.0，推荐 5.x) — 用于高性能仿真和 lint 检查
- **GTKWave** (可选) — 用于查看 VCD 波形文件
- **C++ 编译器** (g++ / clang，支持 C++20) — Verilator 仿真需要
- **make** — Verilator 构建需要

### 安装指南

**Ubuntu / Debian:**

```bash
# Verilator (从源码安装最新版)
sudo apt-get install git perl python3 make autoconf g++ flex bison ccache
git clone https://github.com/verilator/verilator
cd verilator
autoconf
./configure
make -j$(nproc)
sudo make install

# GTKWave
sudo apt-get install gtkwave
```

**macOS (Homebrew):**

```bash
brew install verilator icarus-verilog gtkwave
```

## 编译与仿真

### 使用 Verilator（推荐）

Verilator 将 Verilog 代码编译为 C++ 仿真模型，运行速度快，适合回归测试。

**Lint 检查：**

```bash
verilator --lint-only -Wno-PINMISSING src/rtl/*.v
```

> `-Wno-PINMISSING` 用于忽略 `cp0_write_reg_o` 等有意未连接的端口警告。

**编译并运行仿真：**

```bash
verilator --cc --exe --build -j 0 \
    -CFLAGS "-std=c++20" \
    src/rtl/*.v src/tb/cpu_yellow_top_tb.sv \
    --top-module cpu_yellow_top_tb \
    --timing \
    --trace

./obj_dir/Vcpu_yellow_top_tb
```

**生成覆盖率报告：**

```bash
verilator --cc --exe --build -j 0 \
    --coverage \
    src/rtl/*.v src/tb/cpu_yellow_top_tb.sv \
    --top-module cpu_yellow_top_tb \
    --timing \
    --trace \
    -CFLAGS "-std=c++20"

./obj_dir/Vcpu_yellow_top_tb
```

## 流水线结构

```
    +------+      +------+      +------+      +------+      +------+
    |  IF  | ---> |  ID  | ---> |  EX  | ---> | MEM  | ---> |  WB  |
    +------+      +------+      +------+      +------+      +------+
       |              |              |              |              |
    PC更新        寄存器读取      ALU运算      数据存储器     寄存器写回
    指令取指      控制信号译码   地址计算      访存操作       结果写回
                  分支判断       分支解析
                  数据前递检测
```

### 各阶段说明

| 阶段 | 功能 |
| :--- | :--- |
| **IF** (取指) | 发送 PC 到指令存储器，锁存返回的指令到 IF/ID 段间寄存器 |
| **ID** (译码) | 译码指令、读取寄存器堆、生成控制信号、检测数据冒险、计算分支目标地址 |
| **EX** (执行) | ALU 运算、乘除运算、数据前递选择、分支解析（BEQ/BNE 条件判断） |
| **MEM** (访存) | 数据存储器读写、字节/半字/字对齐处理、CP0 访问、例外检测 |
| **WB** (写回) | 将 ALU 结果/存储器数据/PC+4/CP0 数据写回寄存器堆 |

## 模块说明

### 顶层模块 (cpu_yellow_top)

实例化所有子模块，连接五级流水线的段间寄存器（IF/ID、ID/EX、EX/MEM、MEM/WB）。顶层模块也是处理器对外接口的封装。

### IF 阶段 (if_stage)

管理 PC 更新与指令地址生成：
- 上电复位时 PC = 0xBFC0_0000
- 正常情况：PC_next = PC + 4
- 分支跳转：PC_next = branch_target（来自 ID 阶段）
- 例外处理：PC_next = exception_target（0xBFC0_0380）
- Stall 时 PC 保持不变

### ID 阶段 (id_stage)

指令译码与寄存器读取：
- 根据 opcode 和 funct 字段译码出所有控制信号
- 读取 rs 和 rt 寄存器
- 立即数符号扩展/零扩展
- 计算分支目标地址和跳转目标地址
- 生成前递控制信号

### EX 阶段 (ex_stage)

ALU 运算与数据前递：
- 支持 13 种 ALU 操作：ADD、SUB、AND、OR、XOR、NOR、SLT、SLTU、SLL、SRL、SRA、LUI、PASS_B
- 数据前递多路选择（解决 RAW 数据冒险）
- 乘法/除法运算（MULT、MULTU、DIV、DIVU），结果存入 HI/LO 寄存器
- BEQ/BNE 分支解析与预测错误检测

### MEM 阶段 (mem_stage)

数据存储器访问：
- Load 数据对齐与符号/零扩展（LB、LBU、LH、LHU、LW）
- Store 数据对齐（SB、SH、SW）
- 地址对齐错误检测
- 例外类型判断与传递（Syscall、Breakpoint、Overflow、Reserved Instruction、Address Error）

### WB 阶段 (wb_stage)

写回数据选择：
- 2'b00: ALU 结果
- 2'b01: 存储器读数据
- 2'b10: PC+4（JAL/JALR 返回地址）
- 2'b11: CP0 读数据（MFC0）

### 寄存器堆 (regfile)

32×32 通用寄存器堆，支持双端口异步读、单端口同步写。r0 硬连线为零，写入被忽略。

### ALU (alu)

算术逻辑单元，执行 13 种运算，含溢出检测逻辑（ADD/SUB/ADDI）。

### 冒险检测单元 (hazard_detection)

检测 Load-Use 数据冒险，生成 stall 信号：
- 当 EX/MEM 阶段为 load 指令且目标寄存器匹配 ID/EX 的源寄存器时
- 插入一个 stall 周期（PC 保持、IF/ID 保持、ID/EX 载入 NOP）

### 前递单元 (forwarding_unit)

检测 RAW 数据冒险，生成前递控制信号：
- EX/MEM → EX（1-cycle 前递，优先级高）
- MEM/WB → EX（2-cycle 前递，优先级低）

### CP0 (协处理器0)

系统控制协处理器，管理例外处理：
- 实现 Status、Cause、EPC 三个寄存器
- 例外发生时保存 EPC、设置 ExcCode、跳转到 0xBFC0_0380
- 支持 ERET 指令返回

## 设计特性

- **五级流水线**: IF → ID → EX → MEM → WB
- **57 条 MIPS 指令**: 覆盖算术、逻辑、移位、分支跳转、数据移动、自陷、访存、特权指令
- **数据冒险处理**: 完全的前递(Forwarding) + Load-Use Stall
- **控制冒险处理**: Always Taken 分支预测 + 延迟槽支持
- **CP0 支持**: Status、Cause、EPC 寄存器 + 6 种例外处理
- **存储器**: 哈佛架构，独立指令和数据存储器
- **精确例外**: 例外指令之前的指令全部完成，流水线冲刷后重新取指

### 支持的例外类型

| 例外 | ExcCode | 触发条件 |
| :--- | :--- | :--- |
| Syscall | 5'h08 | SYSCALL 指令 |
| Breakpoint | 5'h09 | BREAK 指令 |
| Reserved Instruction | 5'h0A | 未定义指令 |
| Overflow | 5'h0C | ADD/SUB/ADDI 溢出 |
| Address Error (Load) | 5'h04 | LW/LH 未对齐地址 |
| Address Error (Store) | 5'h05 | SW/SH 未对齐地址 |

### 存储器映射

| 地址范围 | 用途 |
| :--- | :--- |
| 0xBFC0_0000 - 0xBFC0_FFFF | 指令存储器 (64 KB) |
| 0xBFC1_0000 - 0xBFC1_FFFF | 数据存储器 (64 KB) |
| 0xBFC0_0380 | 例外入口地址 |
| 0xBFC0_0000 | 复位入口地址 |

## 自定义与扩展

### 添加新指令

1. 在 `control_unit.v` 中添加新指令的控制信号译码
2. 在 `alu_control.v` 中添加 ALU 操作码（如需要新运算）
3. 在 `alu.v` 中实现新运算逻辑
4. 在 `id_stage.v` 中处理立即数扩展方式（如需要）
5. 在 `mem_stage.v` 中处理访存对齐方式（如为访存指令）
6. 在 Testbench 中添加对应的测试用例

### 修改存储器容量

指令存储器和数据存储器的容量限制在 Testbench 的存储器模型中定义：

- 指令存储器地址映射：PC[31:16] == 16'hBFC0 时选中，局部地址 = PC[15:2]
- 数据存储器地址映射：Addr[31:16] == 16'hBFC1 时选中，局部地址 = Addr[15:2]

修改 `cpu_yellow_top_tb.sv` 中对应存储器数组的大小即可调整容量。

### 调整流水线深度

本项目为标准五级流水线。如需要调整（如增加 EX/MEM 之间的额外流水级），需要：
1. 在 `cpu_yellow_top.v` 中添加新的段间寄存器
2. 调整冒险检测和前递单元的比较逻辑
3. 修改控制信号的流水传递路径

## 常见问题

**Q: Verilator 编译报错 `TIMING` 相关？**
A: 确保使用 `--timing` 选项，并使用支持 C++20 的编译器（`-CFLAGS "-std=c++20"`）。

**Q: 仿真结果全为 FAIL？**
A: 检查存储器初始化是否正确。Testbench 中的指令存储器和数据存储器需要在仿真开始时正确加载初始值。

**Q: VCD 文件过大？**
A: 可以修改 Testbench 中的 `$dumpvars` 参数，仅 dump 感兴趣的信号层级。

**Q: 如何调试单条指令？**
A: 在 Testbench 中找到对应的测试用例，设置断点或使用 `$display` 打印关键信号值。也可以通过波形观察 PC 和寄存器值的变化。
